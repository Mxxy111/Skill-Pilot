# SkillPilot Windows 桌面版

## 运行方式

推荐使用安装版 `SkillPilot-Setup-0.6.0-x64.exe`。安装向导支持选择安装目录，并会创建桌面与开始菜单快捷方式。若不希望安装，可直接运行 `SkillPilot-Portable-0.6.0-x64.exe`。

| | 安装版 | 免安装版 |
|---|---|---|
| 程序形态 | 安装到所选目录 | 单个 EXE，运行时临时展开 |
| 快捷方式 | 自动创建桌面与开始菜单入口 | 不创建 |
| 卸载 | Windows“已安装的应用”中卸载 | 删除 EXE 即可 |
| 功能与数据 | 与免安装版相同 | 与安装版相同 |
| 适合场景 | 长期日常使用 | 试用、U 盘携带或无安装权限环境 |

两者都会把设置、维护清单和备份保存在用户目录，不会把数据嵌进程序文件。因此切换版本不会丢失配置，但复制便携版 EXE 并不会自动带走数据。

桌面版启动后会显示独立窗口，不会打开系统浏览器。点击窗口关闭按钮会隐藏到系统托盘；双击托盘图标可再次打开，托盘菜单的“退出”会完整停止应用及其内置后台服务。

## 数据位置

- 元数据与设置：`%USERPROFILE%\.skillpilot\database.json`
- 停用的 Skills：`%USERPROFILE%\.skillpilot\disabled\`
- 删除前备份：`%USERPROFILE%\.skillpilot\backups\`
- GitHub 来源更新备份：`%USERPROFILE%\.skillpilot\backups\`
- 桌面窗口状态：`%APPDATA%\SkillPilot\window-state.json`

安装版和便携版共用上述数据，因此切换版本不会丢失 Skills 配置。数据库导出不会包含已保存的 API Key。

## 发现与维护的真实边界

- 「发现」先调用 GitHub 搜索；打开仓库后固定默认分支当前 commit，读取文件树并扫描路径、体积、链接类型和已知危险脚本模式。
- 安装时会再次读取同一 commit，只下载所选 Skill 子目录，然后原子写入目标 Agent 目录并记录仓库、commit、子路径和安装位置。
- AI 是可选增强，用于语义推荐和仓库判断；不参与安全放行。AI 未配置或请求失败时，搜索、静态检查和安装仍可使用。
- 自动维护不是占位功能。它只处理来源记录完整的安装项，按仓库合并更新查询，更新前重新扫描，先创建持久备份再原子替换；发现新增高风险、路径消失或扫描不完整时会停止该项并记录错误。
- 计划任务只在 SkillPilot 正在运行或驻留托盘时执行，不是 Windows 系统级后台服务；应用退出后不会运行。
- AI 分类只使用 10 个稳定大类。自动维护不会重复分类内容未变化的 Skill，并且每轮只写一条维护摘要；手动选择 Skill 后分类仍可强制刷新。
- 应用启动后会检查一次最新 GitHub 正式 Release，设置页可手动重新检查。程序只打开发布页，不会在后台替换或执行 EXE。

## 桌面安全边界

- 后台服务仅绑定随机分配的 `127.0.0.1` 端口，不监听局域网地址。
- Electron 渲染进程关闭 Node.js 集成，启用上下文隔离、进程沙箱和 Web 安全策略。
- 所有权限请求默认拒绝，应用导航只能留在启动时分配的本地源。
- 外部链接仅允许无内嵌凭据的 HTTPS 地址，并交由系统默认浏览器打开。
- 应用使用单实例锁，重复启动只会唤回现有窗口。

## 源码运行与打包

```powershell
npm install
npm run desktop
npm run dist:win
```

打包产物位于 `release\`。当前开源构建未附带商业代码签名证书，因此 Windows SmartScreen 可能显示“未知发布者”；公开分发时应使用受信任的代码签名证书签署安装包和便携版。
